# 竞赛题面精修与拆分

## 目标

当用户提供低质量题面时，完成两件事：

1. 在不改变原文语义与信息的前提下，精修文本格式与排版。
2. 将精修后内容拆分为标准文件：
   - `desc.md`
   - `input.md`
   - `output.md`
   - `constraints.md`
   - `sample1.in`, `sample1.out`, `sample2.in`, `sample2.out`, ...

题面来源可为以下任一情况：

- 低质量文本题面（.txt,.preview）
- HTML 文件题面（.htm,.html）
- 低质量文本与 HTML 同时提供（需要交叉核对后统一抽取内容）

## 硬性规则

- **不改内容**：不得新增、删除、改写题意信息；仅允许文字纠错与格式规范化。
- **纠错范围**：修正错别字、错误标点用法、明显排版错误。
- **pangu 间距**：在全角非标点字符与半角非标点字符之间插入空格。
  - 例：`给定n个整数` -> `给定 n 个整数`
  - 例：`输出Yes` -> `输出 Yes`
- **公式规范**（MathJax）：
  - 行内公式使用 `$...$`
  - 行间公式使用 `$$` 包裹，且起止 `$$` 各自独占一行
  - 将 `\(...\)` 包裹的行内公式替换为 `$...$`
  - 将 `\[...\]` 包裹的行间公式替换为 `$$...$$` 形式，且起止 `$$` 各自独占一行
- **数学公式精修**：在不改变数学含义前提下，统一并规范公式写法；可将行内比较关系合并为单个公式片段（如将 `0 ≤ $x$ < $a$` 精修为 `$0 \le x < a$`）。
- **禁止标题**：精修后的题目描述内容中不得出现 Markdown 标题语法（如 `#`、`##`、`###` 开头行）。
- **范围信息归并**：若“题目描述 / 输入格式 / 输出格式”中出现数据范围说明（如 `n \le 2e5`、`$a_i$ 范围`、时间空间限制等），必须移动到“数据范围与提示”对应内容（`constraints.md`）中；移动后原章节不再保留该说明。
- **样例保持原样**：样例输入输出的行内容与换行保持一致，不做语义编辑。
- 范围信息归并示例见 [examples.md](examples.md)。

## 执行流程

复制并维护该清单：

```text
Task Progress:
- [ ] Step 1: 识别原文结构与样例组数
- [ ] Step 2: 精修文本（纠错+标点+pangu+MathJax）
- [ ] Step 3: 结构化拆分为目标文件
- [ ] Step 4: 核对一致性与完整性，并进行格式修正
```

### Step 1: 识别结构

优先识别以下区块（标题可能有同义写法）：

- 题目描述（如“题目描述”“Description”“题意”）
- 输入格式（如“输入”“Input”）
- 输出格式（如“输出”“Output”）
- 数据范围与提示（如“数据范围”“提示”“说明”“Constraints”）
- 样例（如“样例输入/输出”“Sample Input/Output”）

若标题缺失，则按语义段落归类，并在最终结果中保持内容不丢失。

### Step 2: 精修文本

- 只做规范化，不改原意：
  - 错别字修正
  - 标点修正（中文语境优先中文标点）
  - 中英文/数字与中文之间按 pangu 规则加空格
  - 公式转为 MathJax 规范
- 列表、段落、代码块用标准 Markdown 表达。
- 样例块使用 fenced code block，且不改内容。

### Step 3: 文件拆分

按以下映射写入文件：

- 题目描述 -> `desc.md`
- 输入格式 -> `input.md`
- 输出格式 -> `output.md`
- 数据范围与提示 -> `constraints.md`
- 第 `i` 组样例输入 -> `sample{i}.in`
- 第 `i` 组样例输出 -> `sample{i}.out`

样例编号从 `1` 开始，按原文顺序递增。

### Step 4: 一致性检查和格式修正

交付前检查：

- 每个区块是否都已落盘；无对应内容时明确告知用户该文件留空原因。
- `sample*.in` 与 `sample*.out` 数量是否匹配（不匹配需告知）。
- 公式块是否满足 `$$` 独占行。
- 是否存在语义改写风险（如同义替换、删减原句）。
- 将每个区间中 `\(...\)` 包裹的行内公式替换为 `$...$`
- 将每个区块中 `\[...\]` 包裹的行间公式替换为 `$$...$$` 形式，且起止 `$$` 各自独占一行
- 对生成的 `desc.md`、`input.md`、`output.md`、`constraints.md` 执行 `check_text.js` 校验，并确保返回值（exit code）为 `0`。

## 输出格式

向用户汇报时包含：

1. 完成的精修范围（纠错、标点、pangu、MathJax）
2. 生成文件清单
3. 样例文件数量与编号范围
4. 若有歧义或缺失区块，明确说明处理策略

## 术语约定

- “精修”仅指格式与文本质量修复，不含内容改写。
- “数据范围与提示”统一归入 `constraints.md`。
- “样例”指题面中出现的示例输入输出对。
